# 日食模块设计规格

## 目标

新增“日食”学习模块，让用户理解太阳、月球、地球对齐时，本影、半影和伪本影如何决定地面上看到的日全食、日偏食和日环食。

## 核心动线

- 默认进入太空视角，展示太阳、月球、地球、月球影锥，以及地球表面的本影/半影落点。
- 顶部提供四个视角入口：太空视角、日全食、日偏食、日环食。
- “日全食 / 日偏食 / 日环食”按钮是地面观测入口，不是太空视角的过滤器。
- 所有视角共享同一个月球轨迹时间值。用户在任何视角前进、后退、拖动或播放，都会移动同一个月球位置。
- 太空视角中的地球阴影可点击：
  - 点击本影区域进入日全食地面视角。
  - 点击半影区域进入日偏食地面视角。
  - 点击半影不同位置时，地面日偏食的遮挡方向和遮挡深度随点击点变化。
- 日环食作为独立几何案例处理，太空视角显示伪本影/环食带，点击或按钮进入日环食地面视角。
- 太空视角下月球沿地球周围的压缩轨道弦运动，不做脱离地球的垂直平移。
- 地面视角保持正视太阳圆面，不允许用户旋转成侧视薄片。

## 真实机制边界

- 日全食来自月球本影落到地球表面。
- 日偏食来自地面观测点落在半影内；观测点相对本影中心的方向决定太阳被遮挡的一侧。
- 日环食来自月球视直径小于太阳视直径时，伪本影落到地球表面，地面看到太阳边缘形成光环。
- 页面采用教学压缩比例，不追求天体距离和尺寸同时真实，但必须保持相对机制正确。

## 页面结构

- 新增 `/solar-eclipse` 子页面，并在首页模块卡片开放入口。
- 页面顶部保留品牌、返回首页、语言切换。
- 主体为紧凑学习工具布局：
  - 上方视角切换按钮。
  - 中央 3D 主舞台。
  - 右侧状态面板，解释当前视角、当前阶段和点击位置含义。
  - 底部时间控制条，提供后退、前进、播放/暂停、滑杆。

## 技术设计

- 新增纯 TypeScript 几何状态模块，负责把时间、视角、点击点映射为渲染状态。
- 3D 场景使用现有 `@react-three/fiber`、`@react-three/drei`、`three`。
- 太阳、地球、月球模型复用已有真实纹理模型；地球额外叠加日食阴影交互层。
- UI 文案补齐 `src/i18n/dictionaries.ts` 的中英文条目。
- 保持 SSR/测试兼容，浏览器 API 使用前检查环境。

## 验证要求

- `npx tsc --noEmit`
- 相关 `vitest` 测试
- `npm run lint`
- 浏览器验证 `/solar-eclipse` 画布非空、三体可见、阴影可点击、地面偏食角度随点击位置变化。
