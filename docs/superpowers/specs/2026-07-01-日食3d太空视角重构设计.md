# 日食 3D 太空视角重构设计

## 目标

日食模块以太空视角为核心，解释太阳光被月球遮挡后如何形成半影、本影和伪本影，并展示这些影区如何随月球轨道运动扫过地球表面。地球视角作为结果观察区，始终允许用户查看日全食、日偏食和日环食。

## 第一性原理

日食是否表现为日全食或日环食，取决于月球相对地球的视直径和影锥是否到达地球表面：

- 日全食模型：月球视直径大于太阳，本影到达地球，地表出现窄全食带，半影区域出现日偏食。
- 日环食模型：月球视直径小于太阳，本影尖端未到达地球，伪本影到达地球，地表出现窄环食带，半影区域出现日偏食。
- 日偏食不是独立太空模型，而是半影内任意地面观测点看到的结果。

因此太空视角需要二选一模型切换：`日全食模型` / `日环食模型`。地球视角不受该模型限制，始终显示三种地面结果。

## 信息架构

顶层视角：

- `太空视角`
- `地球视角`

太空视角内部：

- `日全食模型`
- `日环食模型`
- 月球轨道时间控制：后退、播放/暂停、前进、慢速/正常速度、进度条

地球视角内部：

- `日全食`
- `日偏食`
- `日环食`
- 同一套时间控制

## 状态模型

将原来的单一 `view` 拆成三个语义字段：

```ts
type MainView = "space" | "ground";
type EclipseModel = "total" | "annular";
type GroundMode = "total" | "partial" | "annular";
```

含义：

- `mainView`：当前显示太空还是地球。
- `eclipseModel`：太空视角当前解释日全食模型还是日环食模型。
- `groundMode`：地球视角当前展示哪种地面观测结果。

## 3D 太空场景

太空视角分四层渲染：

1. 天体层：太阳、月球、地球使用真实 3D 模型和已有真实贴图。
2. 轨道层：月球沿围绕地球的轨道运动，轨道时间驱动月球位置。
3. 影锥层：由太阳边缘和月球边缘的切线生成半影锥、本影锥；日环食模型额外显示伪本影延长区。
4. 地表投影层：影锥与地球白昼面相交，形成宽偏食带，以及窄全食带或窄环食带。影带随轨道时间移动。

## 交互

太空视角影区点击：

- 点击本影带：切换到地球视角并展示日全食。
- 点击半影带：切换到地球视角并展示日偏食；点击位置决定偏食遮挡方向。
- 点击伪本影/环食带：切换到地球视角并展示日环食。

主动进入地球视角：

- 地球视角始终显示三种状态按钮。
- 用户可自由比较日全食、日偏食、日环食。

## 验收标准

- 太空视角使用“太空视角 / 地球视角”顶层导航。
- 太空视角显示“日全食模型 / 日环食模型”二选一。
- 地球视角始终显示“日全食 / 日偏食 / 日环食”。
- 日全食模型中，本影带和偏食带同时出现在地球白昼面。
- 日环食模型中，伪本影/环食带和偏食带同时出现在地球白昼面。
- 月球沿轨道运动时，地球影带位置随时间变化。
- 地球视角不可旋转相机。
- 太阳、月球、地球使用既有真实 3D 模型/贴图体系。
- 中英文文案完整。
- `npx tsc --noEmit`、相关 vitest、`npm run lint` 通过。
- 浏览器截图确认 WebGL canvas 非空，太空视角影锥和地球影带可见。
